# T07: エキゾチックオッズ オーバーレイ検出・EVゲート【Wave 2・T01/T02/T03 依存】

## 目的
T01 で蓄積した発走前エキゾチックオッズ（実測）と T03 の組合せ確率を突き合わせ、
組合せ単位のオーバーレイ（市場合成確率 < モデル確率）を検出して購入候補を生成する。

## 前提条件（着手ゲート）
- T01 配備後 **2〜4週間のデータ蓄積**（最低 200 レース分のエキゾチックオッズ）
- T02 の近似モデル可否判定（近似が使えれば歴史3年で事前スクリーニング、使えなければ実測データのみで前向き検証）

## 中核仮説
単勝市場は効率的（過去検証で確定）だが、組合せ市場は流動性が薄く、
合成オッズの歪み（人気馬絡み過剰買い・穴組合せの過小評価/過大評価）が単勝より大きい可能性がある。
**過去メモリの教訓**: モデルEV単体ゲートは無効。→ EV は必要条件とし、直交バッジ（穴ぐさ/DM/外部指数）× T04 荒れスコアを重ねる。

## 成果物（想定）
- `backend/src/betting/overlay.py` — 合成オッズ vs T03確率の乖離スコア
- `backend/scripts/scan_exotic_overlay.py` — 当日スキャン + 蓄積データでの前向き検証レポート
- T05 ハーネスでの検証（実測オッズ期間は純フォワード扱い）

## 完了判定
ワイド・馬連・三連複それぞれで「オーバーレイ条件×バッジ」の ROI と CI を報告。CI 下限 < 0.85 の券種は破棄。
