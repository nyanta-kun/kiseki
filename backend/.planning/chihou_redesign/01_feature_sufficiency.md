# 地方競馬モデル 特徴量充足性 監査レポート

対象: chihou v9線形 + v10/v11 LightGBM
診断ユニバース: 2025-01〜2026-06, head_count>=6, ばんえい(course=83)除外, 約19.3万馬
測定はすべて上記ユニバースで実測（SQL直）。

---

## ① 現行特徴量の棚卸し

### v9 線形モデル（`chihou_calculator.py`）が出力する5サブ指数
本番v10 LGB はこの5サブ指数を**入力の中核**にしている。各指数が内部で実際に使う生データ:

| サブ指数 | 中身 | 内部で使う生カラム |
|---|---|---|
| `speed_index` | 基準タイム比較・斤量補正 | 走破タイム, distance, weight_carried, condition |
| `last3f_index` | 上がり3F | last_3f系（馬の過去走） |
| `jockey_index` | 競馬場別 過去180日 勝率/連対率 | jockey_id, course, date |
| `rotation_index` | 前走間隔 + 前走着順 | 前走date, 前走着順 |
| `last_margin_index` | 前走1着とのタイム差 | 前走 time差 |

v9 composite は乗数式:
`composite = ability_base(speed.45/last3f.20/last_margin.35) × K_jockey × K_rotation × K_dark × K_margin`
- **K_dark / K_margin の内部で、未使用カラムの一部を“固定係数”として既に消費している**:
  - `running_style`("3"/"4")・`passing_1`(>70%頭数) → 前走後方 +0.10
  - `prev_distance`(短縮200-400m) → +0.05
  - `frame_number`(外枠 ≥0.6×head) → +0.03
  - 騎手変更×上位騎手 → +0.03
  - これらは **LGB特徴量にはなっておらず**、固定の乗数ボーナスとして埋め込まれているだけ（最適化されていない）。

### v10 LGB が使う特徴量（実装上 `ALL_FEATURES` = 17個）
- サブ指数(5): speed_index, last3f_index, jockey_index, rotation_index, last_margin_index
- レースメタ(7): distance, head_count, is_turf, is_dirt, is_good, is_heavy, is_bad
- 馬メタ(5): frame_number, horse_age, weight_carried, horse_weight, weight_change
- アンサンブル: `LGB 0.3 + v9_composite 0.7`
- **注意**: v10 docstring に「追加(3): ext_consensus, prev_distance_diff, running_style_prev」と書かれているが、`ALL_FEATURES` には**含まれていない**（コメントと実装が乖離。実際は17特徴量）。

### v11 LGB（実験）追加4特徴量
improving_form, track_win_rate, class_drop_ratio, prev_pace_ratio(=前走passing_1/head)
- **既知バグ確認済み**: L480 `d["win_probability"] = d["y"]`（ラベルをスコアに代入）→ EV系メトリクス(ev_filter_roi)が**完全に無意味化**。v11のROI評価は信用不可。

### 総括
現行は本質的に「v9が作った能力サマリ5本 + レース/馬の素性10本」しか見ていない。**騎手×厩舎、出走間隔の連続値、距離変動の連続値、コース別の枠・脚質バイアス、相手レベル(相対着差)、クラス連続値**といった“直交情報源”はほぼ全て欠落している。

## ② 未使用カラムのカバレッジ実測表

| カラム | 所属 | 有効値率 | 判定 | 備考 |
|---|---|---:|---|---|
| `trainer_id` | race_entries | **100%** | ✅使える | 厩舎指数の素材。完全充足 |
| `east_west_code` | race_entries | **100%** | ✅使える | 所属区分 |
| `frame_number` | race_entries | 100% | △既使用 | LGBに有るが**コースバイアスとの交互作用**は未活用 |
| `jockey_apprentice_code` | race_entries | **有効14.3%** | ✅使える | 値分布: 0=86%,1〜4で減量区分。**減量騎手は複勝21.8% vs 30.2%** と明確な負バイアス |
| `blinker` | race_entries | 100%だが**全てFalse(ゼロ分散)** | ❌死カラム | 取得されていない。使用不可 |
| `prev_jockey_code` | race_entries | 100%だが**全て`00000`** | ❌死カラム | 乗替り判定に使えない。乗替りは別途導出が必要 |
| `prev_weight_carried` | race_entries | **1.1%** | ❌ほぼ空 | 斤量増減の素材にならない |
| `running_style` | race_results | 100%だが**全て`0`(ゼロ分散)** | ❌死カラム | 脚質ラベルは事実上未取得。脚質はpassingから自前導出必須 |
| `passing_1..4` | race_results | passing_1=**73.8%**, passing_4=高 | ✅使える | 通過順→脚質/位置取りの唯一の生情報源 |
| `time_diff` | race_results | **97.3%(非ゼロ)** | ✅使える | 着差秒。相対着差(相手レベル込み)の素材 |
| `margin` | race_results | **0%(空)** | ❌空 | 使用不可（time_diffで代替） |
| `last_4f` | race_results | **0%** | ❌空 | 使用不可 |
| `place_odds` | race_results | 16.6% | △限定 | 複勝ROI評価用。特徴量にはしない（オッズ原則） |
| `weather` | races | 100% | ✅使える | 晴/曇/雨/雪。conditionと別軸 |
| `grade` | races | 26.1% | △限定 | 重賞のみ |
| `lap_times` | races | 100% | △要パース | テキスト。ペース導出に使えるが前処理コスト |
| `first_3f`/`last_3f_race` | races | **28.6%** | △限定 | レース前後半3F。カバレッジ低、当日値=リーク注意 |
| `prev_distance`(races列) | races | **0%** | ❌空 | races上の列は未充填。前走距離はLATERALで自前取得必要(v9はそうしている) |
| `prev_grade_code`(races列) | races | **0%** | ❌空 | 同上 |
| `post_time` | races | 100% | ✅使える | 発走時刻。ナイター/薄暮の時間帯効果 |
| `race_type_code` | races | 100% | ✅使える | 競走種別 |
| `weight_type_code` | races | 100% | ✅使える | 重量種別(定量/別定/ハンデ) |

**重大な発見**: グラウンディングで「未使用候補」とされた `running_style`・`blinker`・`prev_jockey_code` は**3つともゼロ分散の死カラム**で、そのまま特徴量化しても無意味。脚質・乗替りは生データから**自前で再構築**しなければならない。これは見落とされやすい落とし穴。

## ③ 追加すべき特徴量 優先順位リスト

凡例: 効果=能力/展開推定への寄与見込み, 難度=実装コスト, リーク=情報漏洩懸念。

### 優先度 高（直交情報源・カバレッジ良・リーク低）

1. **厩舎指数 trainer_index（competitor-level）**
   - 根拠: trainer_id 100%充足。地方は厩舎の力差が中央以上に大きく、騎手と相関しきらない直交情報。
   - 期待効果: 大。能力推定の独立軸を1本追加。jockey_indexと同手法(過去180日 競馬場別 勝率/連対率)で実装可。
   - 難度: 中（jockey_indexのロジックを複製）。 リーク: 低（過去成績のみ・当該レース除外で算出）。

2. **前走脚質/位置取り（passing由来, 連続値）**
   - 根拠: running_styleは死カラム。`passing_1/head_count`(=相対前方度, v11 prev_pace_ratio相当)を**連続値特徴量**として正式投入。passing_1有効73.8%。
   - 期待効果: 中〜大。展開推定の中核。「前走後方→今回展開有利」を固定+0.10でなくLGBに最適化させる。
   - 難度: 低。 リーク: 低（前走値）。欠損は-1フラグ。

3. **コース×枠順バイアス（frame × course 交互作用）**
   - 根拠: 実測で課題が顕著。course=50 外枠複勝36.0% vs 内枠26.8%(**+9.2pt**)、course=54 +7.2pt、一方 course=46 は外枠が**負**。コース毎に符号が違うため単純frame_numberでは表現不能。
   - 期待効果: 大。コース別の事前バイアステーブル(過去データ集計の枠別complprate)を特徴量化、もしくは course one-hot × frame の交互作用を許す木を学習。
   - 難度: 中。 リーク: 低（事前集計を当該期間外で作れば無し）。

4. **減量騎手フラグ jockey_apprentice（実測ペナルティあり）**
   - 根拠: 有効14.3%。減量騎手は複勝21.8% vs 非減量30.2%(**-8.4pt**)と明確な負シグナル。jockey_indexだけでは捕捉しきれない（ベテラン乗替り後の見習いなど）。
   - 期待効果: 小〜中。 難度: 低（varchar→減量段階の数値化）。 リーク: 無し。

### 優先度 中

5. **相対着差 / 相手レベル指数（time_diff × レース格）**
   - 根拠: time_diff 97.3%有効。「強い相手に小差負け」を能力として評価する直交軸。last_margin_indexは前走1着差のみで相手の質を見ていない。
   - 期待効果: 中。speed/last_marginの交絡を超える能力推定量になりうる（JRA側の知見=相対走り指数がspeed単体より良かった、と整合）。
   - 難度: 中（レース平均タイム/格でnormalizeが必要）。 リーク: 低（過去走）。
   - 注意: JRA側の教訓では複数seedで効果がseed分散内に消えた前例あり → **5 seed平均A/B必須**。

6. **出走間隔(連続日数)・連闘フラグ**
   - 根拠: rotation_indexは離散スコア化済みだが、地方は連闘・極端な詰めローテが多く、**連続値+連闘0/1**を分離すると非線形を木が拾える。
   - 期待効果: 中。 難度: 低（前走dateとの差日数。v9で取得済みデータ再利用）。 リーク: 無し。

7. **距離短縮/延長（連続値, m差）**
   - 根拠: v9はK_darkで「短縮200-400m→+0.05」と離散固定。連続m差(prev_distance - curr_distance)をLGB特徴量化。
   - 期待効果: 中。 難度: 低（v9のLATERAL前走取得を流用）。 リーク: 低。

8. **クラス変動（賞金/格 連続値, class_drop_ratio相当）**
   - 根拠: v11特徴量だがv11はバグで評価不能。races.prev_grade_codeは空なので**賞金ベース**(prev_prize/curr_prize)で算出。
   - 期待効果: 中。降級馬の妙味は地方の主要edgeの一つ。 難度: 中。 リーク: 低。

### 優先度 低 / 保留

9. **post_time（時間帯・ナイター）** 100%充足だが効果は小さめ。馬場進行(タイム比較)に間接影響。中で。
10. **weather / weight_type_code / race_type_code** one-hotで安価。効果は限定的だが投入コスト低。
11. **lap_times（レースペース）** 100%だがテキストパース+当日値はリーク。前走レースのペースとして使うなら可。難度高で保留。
12. **first_3f/last_3f_race** カバレッジ28.6%・当日値リーク。保留。

## ④ 地方特有で欠落している重要観点

1. **コース別の枠順・脚質バイアスが構造的に未表現**（最大の欠落）。地方は小回り・先行有利が極端で、しかもコースごとに符号が反転する（実測: 同じ外枠でcourse50は+9.2pt、course46は負）。現行はframe_numberを1本の連続値で入れるだけで、コース交互作用を持たない → 地方の最も換金性が高いバイアスを丸ごと取り逃している。

2. **同コース/同距離実績の深さが浅い**。v11 track_win_rateはあるが「3走未満=不明」で薄く、同距離適性は皆無。地方は同一馬が同コースを何度も走るため、過去同条件成績は強い予測力を持つはず。

3. **厩舎軸の不在**。jockey_indexはあるが trainer_index が無い。地方は厩舎の管理力・仕上げ差が大きく、騎手と直交する。

4. **乗替りが検出不能**（prev_jockey_codeが死カラム`00000`）。乗替り＝陣営の本気度シグナルだが、現状は前走entryを別途引いて自前で前走騎手を復元しないと作れない。

5. **相手レベルを織り込んだ能力指標が無い**。last_margin_indexは前走1着とのタイム差のみで、相手が強かったか弱かったかを区別しない。time_diff×レース格で相対能力を作る余地。

6. **出走間隔・連闘の連続表現が無い**（離散スコアのみ）。地方の過酷ローテ文化を捉えきれていない。

## 「能力推定が市場1番人気より劣化」の主因か？（論考）

指数1位 勝率33.5% vs 市場1番人気45.4% という劣化について、**不足特徴量は主因の一部だが単独の主因ではない**と評価する。

- corr(指数rank, 人気)=0.726 と高く、指数は既にオッズと強く揃っている。つまりモデルは「市場が見ている情報の劣化コピー」になっており、**市場が織り込む情報(厩舎の本気度・乗替り・コース適性・調子)を、モデルが持っていないために取りこぼしている**。上記④の欠落（特に厩舎・乗替り・同コース適性）はまさに「市場は見ているがモデルは見ていない」情報で、劣化を一定説明する。
- ただし JRA側の検証教訓（v26_error_analysis）が示すとおり、能力推定の精度向上は「市場1番人気に追いつく」方向には効くが、**戦略原則であるedge(=能力高×市場が人気薄に置く馬)の発掘とは別問題**。特徴量を厚くして市場1番人気に近づけるほど、皮肉にも“オッズに揃う劣化コピー”が高品質になるだけで、回収率は上がらない可能性がある。
- したがって不足特徴量の補完は **(a) 市場が見ている情報（厩舎・乗替り・コース適性）で純粋な精度ギャップを埋める** ことと、**(b) 市場が軽視しがちな情報（コース別枠バイアス・相対着差・減量騎手の過小/過大評価）でオッズの隙間を作る** ことの二系統に分けて評価すべき。劣化解消には(a)が、回収率には(b)が効く見込み。

## 200字要約

現行chihou LGBは「v9由来の能力サマリ5本＋レース/馬素性10本」のみで、厩舎・乗替り・コース別枠脚質バイアス・相対着差・距離/クラス変動といった直交情報を欠く。未使用候補のrunning_style・blinker・prev_jockey_codeは全てゼロ分散の死カラムで脚質/乗替りは自前再構築が必須。有効なのはtrainer_id(100%)・passing_1(74%)・time_diff(97%)・apprentice(減量で複勝-8.4pt)・コース×枠(course50で+9.2pt)。最優先は厩舎指数・コース枠バイアス・前走脚質連続値。市場1番人気への劣化は厩舎/乗替り欠落が一因だが、追加で精度を上げるほどオッズに揃うだけなので回収率向上にはバイアス系edgeの別建てが必要。
