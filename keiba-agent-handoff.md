# 競馬予想システム — 再帰的改善Agent 設計引き継ぎ

## 概要

競馬の指数算出システムにおいて、回収率の継続的向上を目的とした再帰的改善Agentシステムを構築する。

## 現状（確認済み）

- **指数算出**: Pythonスクリプトで動作中
- **データソース**: JRA-VAN DATALAB（SDK経由）
  - Mac上でPython/FastAPIによるProxy経由でアクセス（JRA-VAN SDKはWindows専用のため）
- **実行環境**: MacBook（cc-serverではない）
- **Agent実装**: Claude Code サブエージェント構成

## 未確認事項（Claude Code上で確認すること）

以下はプロジェクトのコードを直接確認して把握する：

1. **現在の特徴量**: スピード指数/血統/騎手・調教師/馬場状態・展開予測のうちどれを使っているか
2. **プロジェクト規模**: 単一スクリプト or モジュール分割済み or DB連携あり
3. **回収率の現状**: 具体的な数値と券種別のバラつき
4. **データの保存形式**: CSV / SQLite / PostgreSQL 等
5. **バックテスト基盤**: 既存の検証コードの有無

## 提案Agent構成（5 Agent + Orchestrator）

### Orchestrator Agent（指揮者）
- 改善サイクル全体の管理
- 停止条件の判定（改善幅が閾値以下 / 過学習検出で巻き戻し）
- 改善履歴の永続化管理
- Claude Code のメインハーネスとして実装

### Agent 1: Feature Engineer Agent（特徴量探索）
- 新しい特徴量の探索・生成・重要度評価
- 不要特徴量の刈り込み（逆方向の動き）
- 例: 上がり3F推移、馬場×血統交互作用、騎手×コース適性スコア

### Agent 2: Strategy Agent（賭け戦略最適化）
- 予測精度とは独立に「どう賭けるか」を最適化
- Kelly基準ベースの資金配分
- 期待値閾値の動的調整（オッズと指数の乖離フィルタ）
- 券種選択の最適配分（単勝/複勝/馬連/三連複）

### Agent 3: Backtester Agent（検証・評価）
- 過去データでのシミュレーション実行
- 評価指標: 回収率、的中率、安定性（シャープレシオ的指標）、最大ドローダウン
- **過学習検出**: 訓練期間と検証期間の分離、検証期間で改善なしならリジェクト
- A/Bテスト（改善前 vs 改善後の比較）

### Agent 4: Analyst Agent（敗因分析・仮説生成）★再帰改善の核
- Backtester結果から「なぜ外れたか」のパターン分析
- 構造的な敗因仮説を生成
  - 例: 「重馬場の逃げ馬を過大評価」「人気薄の追い込み馬を拾えていない」
- Feature Engineer / Strategy Agent への改善指示を生成

### Agent 5: Data Curator Agent（データ品質管理）
- JRA-VANデータの取り込み・クレンジング・欠損値処理・異常値検出
- 鮮度管理（出走取消、騎手変更、馬場変更の反映）

## 改善ループの流れ

```
Orchestrator
  → Backtester（現行モデルで過去N開催を検証）
  → Analyst（敗因パターン分析、仮説生成）
  → Feature Engineer / Strategy Agent（仮説に基づく改善実装）
  → Backtester（改善案をA/Bテスト）
  → Orchestrator（改善なら採用、なければリジェクト → 次サイクルへ）
```

## 設計上の重要原則

1. **指数と賭け戦略の分離**: 予測精度≠回収率。オッズとの乖離（期待値）と資金管理を別Agentにすることで改善の因果関係を明確にする
2. **改善履歴の永続化**: どの仮説を試し結果がどうだったかを全記録。同じ失敗の繰り返しを防止
3. **過学習ガードレール**: 検証期間で改善が見られない変更は自動リジェクト
4. **停止条件**: 回収率改善幅が閾値以下 or 過学習スコア悪化で巻き戻し

## Claude Code での実装方針

- **Orchestrator**: CLAUDE.md またはメインスクリプトからサブエージェントを呼び出すハーネス
- **各Agent**: `claude -p` またはTask toolでサブエージェントとして実行
- **改善履歴**: JSON/YAML ファイルで永続化（例: `improvement_log.json`）
- **設定管理**: 各Agentのパラメータ・閾値を `config.yaml` 等で一元管理
